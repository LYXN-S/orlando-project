# Requirements â€” PostgreSQL Database Integration

## Overview
Integrate a PostgreSQL database into the OrlandoProject Spring Boot backend using a Docker container,
so that all JPA entities are persisted to a real relational database.

---

## Functional Requirements

| ID    | Requirement                                                                                      |
|-------|--------------------------------------------------------------------------------------------------|
| FR-01 | The system **must** provide a PostgreSQL 16 database accessible on **port 5441** (host).         |
| FR-02 | The database must be named `orlandodb` with credentials `postgres / postgres`.                   |
| FR-03 | Hibernate `ddl-auto=update` will auto-create/migrate tables on application startup.              |
| FR-04 | All existing JPA entities must map to PostgreSQL tables correctly on first run.                   |
| FR-05 | Data must survive container restarts (Docker named volume).                                       |

## Tables (auto-generated by Hibernate from existing entities)

| Module    | Entity         | Table Name        | Key Relationships                         |
|-----------|----------------|-------------------|-------------------------------------------|
| Auth      | Staff          | `staff`           | Standalone, soft-deletable                 |
| Catalog   | Product        | `products`        | Standalone, soft-deletable                 |
| Customers | Customer       | `customers`       | Has many Addresses, soft-deletable         |
| Customers | Address        | `addresses`       | Belongs to Customer (`customer_id` FK)     |
| Cart      | ShoppingCart   | `shopping_cart`   | Has many CartItems                         |
| Cart      | CartItem       | `cart_item`       | Belongs to ShoppingCart (`cart_id` FK)      |
| Orders    | Order          | `orders`          | Has many OrderItems, soft-deletable        |
| Orders    | OrderItem      | `order_item`      | Belongs to Order (`order_id` FK)           |

## Non-Functional Requirements

| ID     | Requirement                                                                                  |
|--------|----------------------------------------------------------------------------------------------|
| NFR-01 | Port **5441** must be used to avoid conflict with the host's local PostgreSQL on 5432.       |
| NFR-02 | A single `docker-compose.yml` at the project root to start/stop the database.                |
| NFR-03 | The Spring Boot app connects from the host via `localhost:5440`.                             |
| NFR-04 | No manual SQL scripts needed â€” Hibernate handles schema creation.                            |
| NFR-05 | Sensitive credentials must be easy to override via env vars in the future.                   |
